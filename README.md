# SQL-Power-BI-Business-Revenue-Analysis

## Project Overview
This project demonstrates an end‑to‑end business analytics workflow using SQL and Power BI, starting from raw transactional data and ending with an interactive management dashboard.
The focus is on real‑world data challenges, including inconsistent date formats and currency values stored as text, which are cleaned and standardized using SQL views before visualization in Power BI.

## Business Questions Addressed
- How does revenue evolve over time?
- Which customers contribute the most to total revenue?
- MDoes revenue follow the Pareto (80/20) principle?
- How does current year revenue compare to the previous year (YoY)?

---

## SQL Approach
- Created clean semantic views for analysis:
  - order_items_clean – currency normalization
  - orders_clean – robust date normalization across multiple formats
- Performed:
  - Customer revenue aggregation
  - Pareto (80/20) analysis using window functions
  - Monthly revenue trend analysis
  - Year‑over‑Year (YoY) comparison
    
SQL was used as the primary analytical engine, while Power BI serves as the visualization layer.

## Power BI Dashboard
The Power BI report consists of two focused pages:
### 1. Revenue Overview
- Total revenue KPI
- Average monthly revenue KPI
- Monthly revenue trend
### 2. Customers & YoY Analysis
- Top 10 customers by revenue
- Year‑over‑Year revenue growth
- Monthly YoY revenue comparison
- 
The dashboard is intentionally minimal, emphasizing clarity and business storytelling over visual complexity.
